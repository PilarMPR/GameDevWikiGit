# Applying the Canon — Which Framework for Which Decision

A practical guide for using the wiki's frameworks during actual design work. Not another theoretical summary — this answers: **"I'm facing a design decision right now. Which framework do I reach for?"**

## The decision → framework map

| Design situation | Primary tool | Wiki note |
|---|---|---|
| Defining what the game is *for* | MDA: state the target aesthetics | [mda-framework-overview](../10-Library/notes/mda-framework-overview.md) |
| A playtest felt wrong — diagnosing why | Gulfs of execution/evaluation; flow-exit analysis (boredom vs. anxiety) | [norman-hcd-vocabulary](../10-Library/notes/norman-hcd-vocabulary.md), [flow-channel-definition](../10-Library/notes/flow-channel-definition.md) |
| Designing the core loop | Sellers' 4 principal loops; MDA Dynamics | [game-loops-definition](../10-Library/notes/game-loops-definition.md) |
| Calibrating difficulty | Flow channel + difficulty curve; Koster's boredom test | [flow-channel-definition](../10-Library/notes/flow-channel-definition.md), [tense-and-release](../10-Library/notes/tense-and-release.md) |
| Choosing player count / match structure | Formal Elements (conflict type) + player types | [formal-elements-synthesis](../10-Library/notes/formal-elements-synthesis.md), [bartle-four-types](../10-Library/notes/bartle-four-types.md) |
| Evaluating pacing of a level or session | Interest curve; tense-and-release | [tense-and-release](../10-Library/notes/tense-and-release.md) |
| Designing a level | Teaching function + indirect control; spatial principles | [level-design-principles](../10-Library/notes/level-design-principles.md), [spatial-design-principles](../10-Library/notes/spatial-design-principles.md) |
| Something feels unreadable or confusing | Norman's gulfs + signifier/affordance audit; Hodent's cognitive-load audit | [norman-hcd-vocabulary](../10-Library/notes/norman-hcd-vocabulary.md), [player-psychology-perception](../10-Library/notes/player-psychology-perception.md) |
| Choosing a monetization model | F2P principles + ethical test; SDT undermining check | [f2p-design-foundation](../10-Library/notes/f2p-design-foundation.md), [monetization-models](../10-Library/notes/monetization-models.md) |
| Evaluating whether a mechanic belongs | Elemental tetrad (reinforces theme?); meaningful play (discernible + integrated?) | [formal-elements-synthesis](../10-Library/notes/formal-elements-synthesis.md), [meaningful-play-definition](../10-Library/notes/meaningful-play-definition.md) |
| Player frustration | Tell design + feedback systems; gulf of evaluation | [feedback-system-design](../10-Library/notes/feedback-system-design.md), [norman-hcd-vocabulary](../10-Library/notes/norman-hcd-vocabulary.md) |
| The game isn't retaining players | Outer-loop design + SDT; Koster (patterns exhausted?) | [game-loops-definition](../10-Library/notes/game-loops-definition.md), [player-motivation-maslow](../10-Library/notes/player-motivation-maslow.md) |
| Characters / story aren't landing | Projection vs. empathy; ludonarrative dissonance check | [character-design-projection](../10-Library/notes/character-design-projection.md), [story-narrative-approaches](../10-Library/notes/story-narrative-approaches.md) |

# The Four Principal Loops

The loops a systemic designer must see and shape, from [[../sources/s001-advanced-game-design|Advanced Game Design]]
(s001, ch.7; introduced ch.4).

Loops are "the beating, cycling heart of a game and the player's experience of it" (s001, ch.7).
The four principal loops (s001, ch.7–8):

1. **The game's model** — the game's own internal simulation/system loop.
2. **The player's mental model** — the model the player builds and runs in their head.
3. **The interactive loop** — the player↔game coupling; contains the **core loop(s)**, the
   tight, repeated cycle of action→feedback that defines moment-to-moment play.
4. **The designer's loop** — the designer observing the game + player system and iterating on
   the design, surrounding all of the above.

These build on the general systemic loop types from [[parts-loops-wholes]]:
**reinforcing** and **balancing** loops, plus how resources/currencies move between parts to
create a system (s001, ch.7). Different gameplay loops combine to support the intended player
experience; communicating them uses mockups, prototypes, and design docs (s001, ch.7).

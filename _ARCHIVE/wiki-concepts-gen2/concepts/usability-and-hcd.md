# Usability & Human-Centered Design

The usability vocabulary that game UX is built on, from
[[../sources/s015-design-of-everyday-things|Don Norman, The Design of Everyday Things]] (s015).

## Core concepts (s015)
- **Affordances** (what an object lets you do) and **signifiers** (perceivable cues for where/how to act).
- **Mapping** — natural/spatial correspondence between controls and effects.
- **Feedback** — immediate, informative response to action.
- **Conceptual model / system image** — the model the design conveys so the user understands it.
- **Constraints** — physical, logical, cultural limits that prevent error.
- **Seven stages of action**, bridging the **Gulf of Execution** (how do I act?) and **Gulf of
  Evaluation** (what happened?).
- **Human error is usually design error** — design for error (slips vs mistakes).

## Application to games
- Directly underpins **game UX** ([[../sources/s014-the-gamers-brain|Hodent s014]]) and **player-
  centric interface design** ([[../sources/s006-fundamentals-of-game-design|Adams s006]], ch.12).
- **Feedback** ≈ the **discernable** half of [[meaningful-play]] (s013) and the polish/feedback
  of [[game-feel-and-3cs]] (s007).
- The player's **mental model** = Norman's conceptual model applied to a game ([[fun-and-interactivity]]).

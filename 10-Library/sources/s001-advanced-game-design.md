# s001 · Advanced Game Design (Sellers)

**TL;DR** — A unified approach to game design built on systems thinking: games are systems, and game design is system design. Argues that understanding how parts form loops that create persistent, emergent wholes lets a designer deliberately build games that are "systemic, creative, and engaging" (s001, Intro).
**Provenance caveats** — Summarized from TOC, chapter openers, and author's end-of-chapter summaries; not line-by-line. Citations use chapter numbers.

## Structure (three parts, 12 chapters)

- **Part I — Foundations** (ch.1–4): formal definition of systems (parts/loops/wholes), games as systems, interactivity & fun.
- **Part II — Principles** (ch.5–8): working as a systemic designer — vision/concept doc, game loops, defining game parts.
- **Part III — Practice** (ch.9–12): game balance, teamwork, pitching, prototyping, dev phases, finishing.

## Key chapter claims

- Ch.2 — Formal definition: *"a set of parts that together form loops of interaction between them to create a persistent whole"* (s001, ch.2).
- Ch.3 — Games sit in a nonconsequential **magic circle**; analyzed as structures (tokens/rules), functional elements (loops), and architectural/thematic wholes.
- Ch.4 — Play exists only in the combined **game + player system**; engagement emerges from interactive loops.
- Ch.7 — **Four principal loops**: the game's model, the player's mental model, the interactive loop (core loop), and the designer's loop.
- Ch.9 — Four balancing families: designer-based, player-based, analytical, mathematical; transitive vs intransitive systems.

# Schell's Elemental Tetrad

Every game is composed of four elements that must reinforce each other: **Mechanics** (rules and goals), **Story** (narrative and events), **Aesthetics** (how the game looks and sounds), and **Technology** (the medium of implementation). The tetrad is a design *alignment* tool — when one element conflicts with the others, the game feels incoherent (s005, ch.4).

## Detail

- **Mechanics:** rules, goals, and the algorithms that implement gameplay — what the player does and how the game responds.
- **Story:** the sequence of events; the narrative that unfolds. Linear or emergent. Creates context and meaning for mechanics.
- **Aesthetics:** visual, audio, and haptic design — how the game *presents itself* to the player. Not the same as MDA's Aesthetics (emotional responses).
- **Technology:** the medium — board, screen, VR, phone. Technology creates constraints and affordances for the other three.
- **Alignment principle:** for a game to feel coherent, all four elements must reinforce each other. A dark horror aesthetic with bright puzzle mechanics creates dissonance. A combat-focused mechanic set with no combat narrative loses meaning.
- **Design process:** use the tetrad to audit a design. For each proposed mechanic change, ask: does this conflict with the story, aesthetic, or technology layer? If so, either change the mechanic or update the layer.

> ⚠️ **Tetrad vs. MDA:** Schell's Aesthetics ≠ MDA's Aesthetics. Schell's Aesthetics = visual/audio style. MDA's Aesthetics = emotional responses produced by Dynamics. Schell's Technology has no MDA equivalent.

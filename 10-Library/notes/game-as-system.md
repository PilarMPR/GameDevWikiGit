# Games as Systems — Design as System Design

The foundational premise of Michael Sellers' work: **games are systems, and game design is system design.** A system is a set of parts that interact to form loops that create a persistent whole with properties and behaviors no individual part exhibits alone. This lens is load-bearing across Sellers (s001), Salen & Zimmerman (s013), Fullerton (s008), and MDA (s011).

## Detail

- **System definition (s001, ch.2):** *"A set of parts that together form loops of interaction between them to create a persistent whole; the whole has its own properties and behaviors belonging to the group but not to any single part."*
- **Games are dynamic artifacts (s011):** their content is *behavior*, not media streaming out. The game's system produces behavior when it runs; the designer creates the possibility space.
- **The game+player system (s001):** the player is not just a user of the game — they are *part of the system*. The game changes the player (teaches skills, builds mental model); the player changes the game (drives state changes, exploits emergent interactions). Balance is a property of the game+player system, not the game alone.
- **Systems thinking as design metacognition (s001):** a systemic designer switches between bottom-up (parts→whole) and top-down (whole→parts) views at will; recognizes feedback loops and predicts their effects before playtesting.
- **MDA is a system view (s011):** Mechanics → Dynamics → Aesthetics is a causal chain through the system's layers. Dynamics are the system's run-time behavior; Aesthetics are the system's output to the player.

# Interest Curves — Pacing Across Time

An **interest curve** plots audience interest (y-axis) against time (x-axis) for any entertainment experience. Schell's pacing analysis tool applies at all scales: a 3-minute encounter, a 2-hour session, a 40-hour game. The key structural principle: **the interest curve pattern repeats fractally at multiple scales** (s005, ch.14, Lens #61).

## Detail

- **The successful shape:** hook (early spike earning investment) → rising action (intermediate peaks separated by valleys) → climax (exceeds all previous peaks) → resolution (satisfying conclusion).
- **The hook must come early.** It earns the player's tolerance for the slower teaching that follows. Many games have good content that never gets experienced because the early pacing crosses the interest threshold (the point where the player quits).
- **Valleys are not failures** — they are necessary recovery space that makes peaks feel higher. Without rest, the player habituates to peak tension; without peaks, there's nothing to rest from.
- **Fractal structure:** zoom into any part of a well-paced game and find the same hook-escalation-climax-resolution shape. Each level has its own curve, nested inside the chapter curve, nested inside the game curve.
- **Lens #61 design questions:** Does the experience have a hook? Is there a clear climax that exceeds all previous peaks? Does the curve cross the interest threshold anywhere? Do my intuitions about the curve match playtest data?

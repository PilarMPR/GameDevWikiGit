# Game Balance Methods — Four Approaches

Sellers identifies four complementary approaches to achieving balance. None is sufficient alone; the four work in combination across a game's lifecycle (s001, ch.9).

## Detail

| Method | Description | When to use |
|---|---|---|
| **Designer-based** | Intuition and judgment from experience | First approximation; always present |
| **Player-based** | Observing how players actually behave | Playtesting; real-behavior data |
| **Analytical** | Data and analytics from the running game | Live games; post-launch tuning |
| **Mathematical** | Probabilities, models, expected values | Economy design; randomness systems |

- **Schell's methodological principles (s005, ch.11):** Double-and-halve (don't change values in small increments — double or halve to explore the design space quickly); define the problem before proposing solutions (many cascading balance errors come from fixing the wrong thing); playtest with a skill mix (developer team is too expert; mixed-skill reveals the real difficulty curve).
- **Transitive balance:** assign numerical value to each attribute, sum per option, adjust until totals equivalent. Then validate with playtesting — the model informs the prototype; the prototype informs the model.
- **Balance is never done:** any player can discover new imbalances. Live games balance continuously. Design resilient systems, not perfect ones.

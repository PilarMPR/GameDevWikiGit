# Lens #90 — Documentation

Design documentation exists to capture decisions and communicate the current state of the design. Out-of-date documentation is worse than no documentation — it misleads. (s005, ch.24)

## Core Questions

> Does documentation reflect the current state? Is it being used? Is there a single source of truth? Are decisions captured before they're forgotten?
